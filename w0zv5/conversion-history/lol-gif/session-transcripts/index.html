PHASE 2: TIMING ANALYSIS (DO THIS FIRST!)

2.1 Extract Frame Delays

# Get individual frame delays in milliseconds
identify -format "Frame %s: %Tms\n" input.gif

# Or get detailed info
identify -format "Frame %s: delay=%T (0.01s units) = %Tms\n" input.gif

2.2 Calculate Animation Duration

# Sum all delays
identify -format "%T\n" input.gif | awk '{sum+=$1*10} END {print sum "ms"}'

Critical checks: - ‚úÖ Is Frame 0 longer than others? (common for "hold at start") - ‚úÖ Are all frames uniform, or do some hold longer? - ‚úÖ What's the total duration in seconds?

2.3 Calculate Keyframe Percentages

For SVG @keyframes, you need percentages, not milliseconds:

For each frame N:
  - cumulative_time = sum of delays from frame 0 to N
  - percentage = (cumulative_time / total_duration) √ó 100

Example (no.gif): - Frame 0: 0ms ‚Üí 1000ms (0%, 32.3%) - Frame 1: 1000ms ‚Üí 1100ms (32.3%, 35.5%) - Frame 2: 1100ms ‚Üí 1200ms (35.5%, 38.7%) - ...and so on

Common mistake: Assuming uniform percentages (0%, 4.5%, 9.1%, ...) when Frame 0 has a special delay!

PHASE 3: MOVEMENT ANALYSIS (PIXEL-LEVEL MEASUREMENT)

3.1 Create Diff Images

What to look for: - Red pixels = what changed - Black pixels = what stayed the same - No red on face outline? ‚Üí Face circle is static ‚úì - Red only on features? ‚Üí Only features move ‚úì

3.2 Identify What Moves vs. What's Static

Questions to answer: - Does the face circle outline change? (should be NO for most smileys) - Do eyes move? Change shape? Change size? - Does mouth move? Change shape? - Is there compression/stretching, or just translation?

Method: Look at diff images at high magnification - If outline has red pixels: it's changing - If outline is black: it's stable (good!)
